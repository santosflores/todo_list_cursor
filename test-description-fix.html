Task Description Preservation Fix Test

Test Case: Moving Task Between Columns

Expected Behavior: When a task with a description is moved from one column to another, the description should be preserved.

Previous Issue: Task descriptions were being erased when moving tasks between columns.

Fix Applied: Modified the batchUpdateTasks and updateTask methods to only update fields that are explicitly provided in the update, preserving existing values for fields not included in the update.

✅ Fix Summary

Files Modified:

  • src/app/services/task.service.ts - Lines 637-638 and 330

Changes Made:

  • Changed description: update.updates.description?.trim() to description: update.updates.description !== undefined ? update.updates.description.trim() : updatedTasks[taskIndex].description
  • Changed description: updates.description?.trim() to description: updates.description !== undefined ? updates.description.trim() : t.description

Root Cause: When moving tasks between columns, only the order and status fields were being updated, but the code was still trying to process the description field. Since update.updates.description was undefined, it was setting the description to an empty string.

Solution: Only update fields that are explicitly provided in the update object, preserving existing values for fields not included in the update.

How to Test

  1. Start the application: npm start
  2. Create a task with a description
  3. Drag the task from one column to another
  4. Verify that the description is preserved after the move
